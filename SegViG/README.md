# SegViG: Segment-Aware Vision GNN for Walkability Prediction

This repository contains the implementation of **SegViG**, an end-to-end walkability prediction framework that integrates objective urban form with subjective pedestrian perception via a segment-aware Vision Graph Neural Network.

## Quick Start: Reproduce Results

**1. Download dataset, checkpoint, and pre-trained weights**

- Dataset (images + CSV files): **[https://github.com/anonymous94949/Dataset](https://github.com/anonymous94949/Dataset)**
- Trained checkpoint: **[LINK](https://drive.google.com/file/d/1xjt3Nl6ryGNep98U_a-30NZ-2HI7tSKG/view?usp=sharing)**
- ViG-B pre-trained weights: **[LINK](https://drive.google.com/file/d/1uVJVIzEUFgicuNBJfgE6IpNefbZseTVY/view?usp=sharing)**

**2. Extract and place files**

```bash
# Extract checkpoint
tar -xzf checkpoints.tar.gz -C /path/to/SegViG/
```

Place files as follows:

```
SegViG/
├── checkpoints/
│   ├── last.pth.tar
│   └── args.yaml
├── pretrained_model/
│   └── vig_b_82.6.pth     # downloaded from ViG repository
├── meta_data/
│   ├── train.csv          # downloaded from dataset repo
│   ├── test.csv
│   └── segment_overlap/   # included in this repo
│       └── train_test.json
└── images/                # downloaded from dataset repo
```

**3. Run inference**

```bash
bash run_inference.sh
```

> **To train from scratch**, see the [Training](#training) and [Data Preparation](#data-preparation) sections below.

## Data Preparation

### Dataset Download

Download the dataset (images + CSV files) from:
**[https://github.com/anonymous94949/Dataset](https://github.com/anonymous94949/Dataset)**

The segment maps (`train_test.json`) are included in this repository under `meta_data/`.

### Directory Structure

Place the downloaded files as follows:

```
SegViG/
├── pretrained_model/
│   └── vig_b_82.6.pth
├── meta_data/
│   ├── train.csv          # downloaded from dataset repo
│   ├── test.csv
│   └── segment_overlap/   # included in this repo
│       └── train_test.json
└── images/                # downloaded from dataset repo
    └── *.jpg / *.png
```

### CSV Format

`train.csv` / `test.csv`:

| Column | Description |
|--------|-------------|
| `merged_name` | image filename (e.g., `scene_001.jpg`) |
| `avg_q7` | walkability label, 1-based integer (1–4) |

### Segment Data Format

`train_test.json` maps each image filename to a 14×14 segment ID array (patch-level, generated by Mask2Former):

```json
{
  "scene_001.jpg": [[0, 0, 1, ...], [1, 1, 2, ...], ...],
  ...
}
```

Each entry is a list of 14 lists, each containing 14 integer segment IDs corresponding to the 14×14 patch grid of the input image.

## Training

```bash
bash run.sh
```

Or manually:

```bash
python train.py \
  --training_mech vig \
  --model vig_b_224_gelu \
  --pretrained \
  --pretrain_path ./pretrained_model/vig_b_82.6.pth \
  --sched cosine --epochs 50 --opt adamw --batch-size 64 \
  --warmup-lr 1e-6 --model-ema --model-ema-decay 0.99996 \
  --aa rand-m9-mstd0.5-inc1 --color-jitter 0.4 \
  --warmup-epochs 20 --opt-eps 1e-8 --remode pixel --reprob 0.25 \
  --amp --lr 2e-3 --weight-decay 0.05 --drop 0 --drop-path 0.1 \
  --meta_dir ./meta_data \
  --no-prefetcher \
  --data_dir /path/to/images \
  --num-classes 4 \
  --corn_loss --smoothing 0 \
  --use_segmentation_edge knn_in_seg \
  --contrastive_loss --num_knn 18 --overlap \
  --eval-metric top1
```

### Key Arguments

| Argument | Description | Default |
|----------|-------------|---------|
| `--use_segmentation_edge` | Graph construction mode: `knn_in_seg` / `seg_knn` / `seg_only` / `False` | `False` |
| `--corn_loss` | Use CORN ordinal regression loss (L_CLS) | `False` |
| `--contrastive_loss` | Add self-supervised ordinal loss L_Rank | `False` |
| `--contrast_alpha` | Weight λ for L_Rank | `0.1` |
| `--num_knn` | Number of nearest neighbors | `18` |
| `--overlap` | Use overlapping segment maps | `False` |
| `--training_mech` | `vig` (single image) / `vig_pano_feat` / `vig_pano_pred` (panoramic) | `vig` |

## Graph Construction Modes

| Mode (`--use_segmentation_edge`) | Paper Name | Description |
|----------------------------------|------------|-------------|
| `knn_in_seg` | **KNN in Seg** (ours) | kNN search restricted to same-segment nodes via distance masking |
| `seg_knn` | **Seg KNN** | kNN search restricted to same-segment nodes via index replacement |
| `seg_only` | **Seg Only** | Connect all nodes within the same segment (no kNN ranking) |
| `False` | **KNN** | Standard feature-space kNN without segmentation constraints |
